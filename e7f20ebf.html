超微 X10DAi安装黑苹果10.13.6 OC引导 P4000独显

发表于 2020-12-05 更新于 2021-12-27 分类于 台式机 ， Supermicro_超微
配置

处理器 英特尔 Xeon(至强) E5-2650 v4
主板 超微 X10DAI ( Wellsburg )
内存 64 GB ( 三星 DDR4 2133MHz )
主硬盘 Lexar 500GB SSD ( 500 GB / 固态硬盘 )
主显卡 Nvidia Quadro P4000 ( 8 GB / Nvidia )
显示器 惠科 HKC2701 T7000 DO ( 27 英寸 )
声卡 瑞昱 ALC888 @ 英特尔 Wellsburg 高保真音频
网卡 英特尔 I210 Gigabit Network Connection / 超微

配置分析

CPU：Xeon(至强) E5-2650 v4

速度 2.20 GHz (100 MHz x 22.0)
处理器数量 核心数: 12 / 线程数: 24
核心代号 Broadwell-E

单个CPU，Broadwell-E核心，OC引导可按HEDT配置来做。

主板：超微 X10DAI

主板型号 超微 X10DAI
芯片组 Wellsburg
序列号 NM173S017916
主板版本 1.02
BIOS版本 American Megatrends Inc. 3.2 / BIOS程序发布日期: 12/16/2019

超微X10DAI，之前己经用CLOVER安装过，这次选用OC，估全部重新来过。需要注意，主板需要升级为最新的3.2版本BIOS，超微X10DAI是X99芯片组。

显卡：Nvidia Quadro P4000

专业卡，但N卡只能安装macos 10.13.6。

声卡，网卡，USB

声卡是ALC888,APPLEALC ID为1即可，但仍需要用SSDTTIME生成SSDT-HPET.aml，否则认不出声卡。网卡有个万兆的，有两个千兆的，都可以驱动，USB需要注放X99的USB KEXTS，但一般来说够用。

引导，系统版本
  • OPENCORE 5.9
  • MACOS high sierra 10.13.6

BIOS设置

Above 4G 开启，这个BIOS没有CFG可以关，也没有CSM选项。

常见错误

1，卡在end randomseed

没有正确地填入FAKECPUID

2，卡在please go to https://panic.apple.com to report this panic

超微x10dai加Broadwell-E核心配置最常见的错误，这个要根据系统的版本相应地仿冒CPU ID，否则系统是不认这个CPU的。
