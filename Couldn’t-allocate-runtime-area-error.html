黑苹果安装出现Couldn’t allocate runtime area错误的另一种解决思路

发表于 2019-10-29 更新于 2019-11-15 分类于 教程

错误表现为CLOVER选择macos进去就，马上跳出以下错误。
Error allocating 0x0x11cb4 pages at 0x000000000139c2000 alloc type 2

多数由内存引起，尤其在一些笔记本，如蓝天模具的神舟笔记本上特别容易出这样的错误。

一般网上的教程都建议计算Slide值，并在boot args中添加slide=计算到的Slide值。
可以参考：https://github.com/wmchris/DellXPS15-9550-OSX/blob/master/Additional/slide_calc.md

但这种方法的确有点深奥，还要涉及16进制计算。

而老吴我的方法，就比较直接一点，既然是内存引起的，那直接找相关的补丁就好了。

第一，先到我这边下载相关的内存补丁：黑苹果卡加号的常见解决方之一 OsxAptioFixDrv | 老吴黑苹果工作室
一共5个，其中，AptioMemoryFix-64.efi比较常用，但如果遇到上面的错误，我就建议换成OsxAptioFix2Drv-free2000.efi，切记。

第二，boot args中还是添加一条slide=1，注意，是等于1，至于为什么是1，完全经验所得。

第三，在机型smbios里，注入你的机子内存信息，多少插槽，多大容量，什么品牌，频率，最好填上，填对。

通过以上三个方法，我的确解决了不少这样的错误，大家可以参考一下，经验所得，毫无保留。
